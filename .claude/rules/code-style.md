# コードスタイルと規約

## 基本設定

- **インデント**: スペース2つ
- **行幅**: 100文字
- **引用符**: シングルクォート
- **末尾カンマ**: ES5
- **改行コード**: 自動検出

## TypeScript設定

- 厳密な型チェック有効
- パスエイリアス: `@/` → `src/`ディレクトリ
- 未使用変数: アンダースコア `_` で始まる変数は許可

## インポート順序

1. builtin (Node.js組み込み)
2. external (外部ライブラリ)
3. internal (プロジェクト内)

- Reactは最優先でインポート
- アルファベット順（大文字小文字区別なし）

## 命名規則

- コンポーネント: PascalCase
- ファイル名: PascalCaseまたはkebab-case
- 変数・関数: camelCase
- 定数: UPPER_SNAKE_CASE
- 型・インターフェース: PascalCase

## セキュリティ

- シークレットや鍵をコードに含めない
- 環境変数で機密情報を管理（`src/env.ts`でバリデーション）
- `.env`ファイルはコミットしない
